### 🔥 격차 #1 — 미래 날짜 자동 잠금 (운영 안정성)

**시트**: 사용법 탭에 명시:
> "현재 날짜 기준 일주일 후 날짜의 열이 모두 잠깁니다."
> "다음달 5일이 지나면 이전달 출석부가 숨김&수정불가가 됩니다."

**효과**:
1. 미래 1주일 이후 날짜 셀 → 입력 불가 (실수 방지)
2. 정산 마감 후 (다음달 5일) 이전달 셀 전부 잠금 (정산 후 수정 차단)

**우리 앱 현황**: 무제한 수정 가능. 운영자가 실수로 과거 정산을 건드릴 위험.

**구현 방안**: AttendanceTable 에 `lockedDateSet` 계산 → 셀 클릭 시 차단 + 이전달 5일 이후엔 전체 disabled. 관리자 토글로 "잠금 해제" 가능.

**작업량**: M (반나절)

### 🟡 격차 #8 — 일별 시간표 / 강의실·교시별 학생 배정

**시트**: "시간표" 탭 — 강의실 × 교시 × 날짜 매트릭스
- 강의실: 프리미엄관 / 의치대특별반
- 교시: 1교시 (2:30~4:20), 2교시 (4:20~6:10) 등
- 셀: 학생 이름/학교 (예: "박지율/6침산초")
- 색상: 빨강 = 결석/이슈, 초록 = 특이사항

**우리 앱 현황**: enrollment 의 days 만 있음. 강의실/교시/일자별 학생 배정 데이터 없음.

**구현 방안 (큰 작업)**:
1. `class_schedule` 테이블 신설: (date, classroom, period, student_id, status, memo)
2. 시트 시간표 → DB 동기화 sync
3. 신규 페이지 `/schedule` — 일별 강의실 × 교시 매트릭스 뷰
4. 출석부에서 시간표 데이터 인용 (보강/결강 자동 매칭)

**작업량**: L~XL (1~2주)

**가치**: 매우 큼 — 시트의 핵심 운영 도구. 다만 우리 앱 범위 확장으로 별도 phase 권장.
